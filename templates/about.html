How did this app came into existence?

Once the Covid-19 lockdown around the world eases (hopefully sooner than later), some of you will be in charge of gradually bringing the staff back into the office. But how do you keep track of everyone coming to work and if the maximum number of colleagues in the office for a given day has been reached? I believe there is an easier way to centralize and keep track of every staff member that will come to the office than just counting the email notifications the empployees will be asked to send or try to use a shared Excel solution that comes with its own drawbacks.

You could indeed ask your colleagues to send an email with the days when they will be present in the office, or you could delegate this activity to each team's manager. Regardless, someone has to do the counting, and the bigger the organization, the harder it gets to keep track of everyone, and it's only Monday. You could try to use shared Excel workbooks saved on shared drives, OneDrive, SharePoint or even use Excel Online as an alternative, but only the ones who've used these solutions know what hassle it is to have a large number of users using the same file at the same time. From users editing the same cell, locked out users while the file is in use, corrupted or missing data, to users hiding or moving columns, formulas, there are a lot of things that can go wrong.

As a personal programming project while trying to learn Flask, I've built this website for keeping track of staff returning to the office. The employees add the presence days in the online form, and the employee count for each day is visible immediately for everyone. You no longer have to keep track of each staff member coming into the office, and the days where the threshold of employees has been reached are being highlighted. As an admin, on top of what a regular user can do, you can change the maximum number of employees in the office (maybe your office capacity can support 100 employees in the office at any point instead of 20) and you can even delete the entries if you no longer need them.

Because I believe that this idea can be useful and I'm sure that improvements and other functionalities can be added to it, I've uploaded the complete code to this public GitHub repository. If you're interested, feel free to fork the project, play with it, break it, add your own personal touch, and maybe even implement this idea - or a similar one - in your organisation.
